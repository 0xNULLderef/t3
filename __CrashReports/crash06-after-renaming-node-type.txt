Crash after renaming Node type in rider
---------------------

1. I had a compiling version of a node called "GetSrvSize"
2. Close T3
3. Refactor to "GetTextureSize"
4. Rename .t3 and t3ui files to match new name
5. Start t3
---> Crash



work-around rename both settings in .t3 file:

```
{
  "Name": "GetTextureSize",
  "Id": "064c67af-68bc-4bd3-91f4-6a85388b41ed",
  "InstanceType": "T3.Operators.Types.GetTextureSize, Operators",
  "Inputs": [
    {
      "Id": "8b15d8e1-10c7-41e1-84db-a85e31e0c909"/*Texture*/,
      "DefaultValue": null
    }
  ],
  "Children": [],
  "Connections": []
}
```

This is super annoying, because renaming types is a frequent action. Ideally this should be possible from t3.
